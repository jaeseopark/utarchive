# PR 22 — Cover Art Support

**Phase:** 2 — Core REST API / UI  
**Depends on:** PR 02, PR 07, PR 08, PR 14

## Goal

Add a dedicated cover art table and allow songs and albums to optionally reference cover art. Render the appropriate cover art for songs by falling back through parent songs, album associations, and album cover art.

## Scope

**In:**
- New `cover_art` table storing image metadata and file path
- Optional `cover_art_id` on songs and albums
- Album API accepts/returns `coverArtId`
- Song API accepts/returns `coverArtId`
- Song detail UI displays song cover art if available, otherwise traverses the family tree:
  1. use the song's own `coverArtId`
  2. if absent, use the album cover art of any associated album(s)
  3. if still absent, check the parent song recursively until a cover art is found or the master song is reached
- No image upload UI yet; cover art can be attached through API or admin tooling later

## Database Schema

| Table | Notes |
|---|---|
| `cover_art` | `id`, `file_path`, `width`, `height`, `file_extension`, `file_size_bytes`, `created_at` |
| `songs` | add `cover_art_id uuid` |
| `albums` | add `cover_art_id uuid` |

## Request / Response Schemas

### Song create/update body
```ts
{
  title: string,
  parentId?: string | null,
  platformId?: string | null,
  releasedAt?: string,
  artistIds: string[],
  url?: string,
  filePath?: string,
  coverArtId?: string | null,
  description?: string,
  preferred?: boolean,
  trimStart?: number | null,
  trimEnd?: number | null
}
```

### Album create/update body
```ts
{
  title: string,
  artistId: string,
  yearReleased?: number,
  coverArtId?: string | null,
  trackList?: Array<{ number: number, title: string }>,
  urls?: Record<string, string>
}
```

### Cover art response object
```ts
{
  id: string,
  filePath: string,
  width: number,
  height: number,
  fileExtension: string,
  fileSizeBytes: number,
  createdAt: string,
}
```

## UI behavior

On the song detail page:
- Prefer the song's own cover art
- If missing, check albums containing the song and use the first album cover art found
- If still missing, walk the parent chain and repeat the lookup on each parent song
- If no cover art is found, fall back to a generic placeholder image

## Acceptance Criteria

- [ ] `cover_art` table exists and stores metadata for image files
- [ ] Songs and albums can optionally reference `coverArtId`
- [ ] Song detail response includes a resolved `coverArtId` if the song has its own cover art or fallback art found from parent/album associations
- [ ] Song detail page displays the resolved cover art image
- [ ] Parent-song fallback logic is exercised in tests for a three-level family chain
- [ ] If no cover art exists anywhere, the UI shows a placeholder image
