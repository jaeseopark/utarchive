# PR 01 — Monorepo Scaffold

**Phase:** 1 — Foundation  
**Depends on:** none

## Goal

Establish the repository structure, tooling, and shared configuration so every subsequent PR builds on a consistent foundation.

## Scope

**In:**
- npm workspaces monorepo with `frontend/` and `backend/` packages
- Shared `.nvmrc` pinned to Node LTS at repo root
- Root `package.json` with workspace definitions and shared dev scripts
- TypeScript config hierarchy: root `tsconfig.base.json`, extended by each package
- ESLint + Prettier config shared across packages
- `.env.example` documenting all required environment variables
- `.gitignore`

**Out:**
- No application code (routes, components, DB)
- No Docker setup (PR 05)
- No dependency installation beyond tooling

## Key Files / Directories

```
.nvmrc
.env.example
.gitignore

tsconfig.base.json
eslint.config.js

  package.json                (empty scaffold; deps added in PR 03)
  tsconfig.json               (extends ../../tsconfig.base.json)
```

## Acceptance Criteria

- [ ] `node --version` matches `.nvmrc` after `nvm use`
- [ ] `npm install` from repo root installs all workspace deps without errors
- [ ] `npx eslint .` runs without crashing (zero files linted is fine at this stage)
- [ ] Both `frontend/tsconfig.json` and `backend/tsconfig.json` compile without errors (`tsc --noEmit`)
