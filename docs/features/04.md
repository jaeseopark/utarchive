# PR 04 — Auth API

**Phase:** 1 — Foundation  
**Depends on:** PR 02, PR 03

## Goal

Implement login, logout, and authentication middleware. Only authenticated requests may access any route except `/health` and `/auth/login`.

## Scope

**In:**
- `POST /auth/login` — validates id + password + TOTP code, issues JWT
- `POST /auth/logout` — clears the JWT cookie
- `GET /auth/me` — returns the authenticated user identity (for frontend session check)
- Auth middleware that validates the JWT on every protected route
- TOTP validation using the `otpauth` library against the `totp_key` from `AUTH_CREDENTIALS`

**Out:**
- No multi-user support — credentials are always read from the single `AUTH_CREDENTIALS` env var
- No refresh tokens
- No rate limiting (can be added later)

## Auth Flow

1. Client sends `POST /auth/login` with `{ id, password, totpCode }` in request body.
2. Server parses `AUTH_CREDENTIALS=id,password,totp_key` from env.
3. Validates `id` and `password` with constant-time comparison.
4. Validates `totpCode` against the TOTP key using `otpauth`.
5. On success: signs a JWT `{ sub: id }` with `JWT_SECRET`, sets it as an `httpOnly; Secure; SameSite=Strict` cookie named `session`, TTL = `JWT_TTL_SECONDS`.
6. Auth middleware on all other routes reads the cookie, verifies the JWT, and attaches the payload to `req.user`. Returns 401 on failure.

## Key Files / Directories

```
backend/
  src/
    routes/
      auth.ts              (POST /auth/login, POST /auth/logout, GET /auth/me)
    middleware/
      requireAuth.ts       (JWT cookie validation; attaches req.user)
    lib/
      totp.ts              (TOTP validation wrapper around otpauth)
      jwt.ts               (sign / verify helpers)
```

## Environment Variables Used

| Var | Purpose |
|---|---|
| `AUTH_CREDENTIALS` | `id,password,totp_key` |
| `JWT_SECRET` | HMAC signing key |
| `JWT_TTL_SECONDS` | Token lifetime (default 3600) |

## Acceptance Criteria

- [ ] `POST /auth/login` with correct id + password + valid TOTP returns 200 and sets `session` cookie
- [ ] `POST /auth/login` with wrong password returns 401
- [ ] `POST /auth/login` with wrong TOTP code returns 401
- [ ] `GET /health` returns 200 without a cookie
- [ ] Any other route without a valid `session` cookie returns 401
- [ ] `POST /auth/logout` clears the cookie and returns 200
- [ ] `GET /auth/me` with valid session returns `{ id }` and 200
