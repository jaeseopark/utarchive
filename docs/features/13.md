# PR 13 — Artist Pages

**Phase:** 3 — Frontend Shell  
**Depends on:** PR 06, PR 11, PR 12

## Goal

Build the artist list page and artist detail page. These are read-only views; artist creation/editing is out of scope for this PR.

## Pages

### Artist List — `/artists`

- Paginated table of all artists with columns: Name, Aliases, Song Count (from `GET /api/artists`)
- Each row links to the artist detail page
- Simple pagination controls (Previous / Next)

### Artist Detail — `/artists/:id`

- Artist header: name, aliases, description
- URL map rendered as a list of labeled links (e.g. "YouTube →", "Twitter →")
- Song list: table of all songs associated with this artist
  - Columns: Title, Released, Preferred, Play Count
  - Each song title links to `/songs/:id`
  - Sorted by `releasedAt` descending by default

## Key Files / Directories

```
frontend/
  src/

      ArtistDetail.tsx
    components/
      SongTable.tsx      (reusable song list table; also used in PR 14+)
      UrlMap.tsx         (renders a jsonb urls map as labeled links)
```

## Acceptance Criteria

- [ ] Artist list shows all artists, paginated at 50 per page
- [ ] Clicking an artist navigates to the detail page
- [ ] Artist detail shows name, aliases (comma-separated if multiple), description, and URLs
- [ ] `urls` map with zero entries renders nothing (no empty list)
- [ ] Song table on artist detail is sorted by released date descending
- [ ] Each song title in the table links to the correct `/songs/:id` route
- [ ] Loading states show a skeleton or spinner while API calls are in flight
- [ ] Error state shows a user-friendly message if the API call fails
