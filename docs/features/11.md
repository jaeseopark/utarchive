# PR 11 — React App Scaffold

**Phase:** 3 — Frontend Shell  
**Depends on:** PR 01

## Goal

Set up the React frontend with all tooling, a base layout, and a typed API client. No business-logic pages yet — just the scaffold every subsequent frontend PR builds on.

## Scope

**In:**
- Vite + React + TypeScript
- Tailwind CSS + shadcn/ui initialization (`npx shadcn@latest init`)
- React Router v6 with a root route layout (nav sidebar + content area)
- Typed API client: a thin `fetch` wrapper that attaches the `session` cookie, handles 401 → redirect to login, and parses JSON responses
- Placeholder pages for every route (enough to verify routing works)
- Zod schemas for all API response shapes (mirroring the backend Zod schemas)

**Out:**
- No real page content (subsequent PRs)
- No auth (PR 12)
- No audio player (PR 17)

## API Client Conventions

```ts
// Usage:
const artist = await api.get('/api/artists/123', ArtistSchema);
const created = await api.post('/api/artists', body, ArtistSchema);
```

- On 401: clears local session state and redirects to `/login`
- On non-2xx: throws a typed `ApiError` with `{ status, message }`
