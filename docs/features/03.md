# PR 03 — Express Server Core

**Phase:** 1 — Foundation  
**Depends on:** PR 01

## Goal

Stand up the Express + ws server with shared middleware, error handling, and a health-check endpoint. This is the skeleton that all API route PRs attach to.

## Scope

**In:**
- Express app with TypeScript
- `ws` WebSocket server attached to the HTTP server (wired up now; no business logic yet)
- Zod-based request validation middleware (reusable helper for body/query/params)
- Global error handler (maps Zod errors → 400, unhandled → 500)
- `GET /health` endpoint
- Environment variable validation on startup (using Zod) — crash fast if required vars are missing
- Graceful shutdown handler (SIGTERM / SIGINT)

**Out:**
- No auth (PR 04)
- No database queries (PR 02 schema is ready but routes come per-feature)
- No static file serving (PR 05 / Dockerfile)
- No WebSocket message handlers yet

## Key Files / Directories

```
backend/
  src/
    index.ts               (entry point: creates server, attaches ws, starts listening)
    app.ts                 (Express app factory — separates app from server for testing)
    middleware/
      validateRequest.ts   (Zod schema → Express middleware factory)
      errorHandler.ts      (global error handler)
    routes/
      health.ts            (GET /health)
      index.ts             (router registry)
    config.ts              (env var parsing + Zod validation)
    ws/
      index.ts             (WebSocket server setup; placeholder broadcast helper)
```

## Environment Variables Used

| Var | Purpose |
|---|---|
| `PORT` | Listening port (default 3000) |
| `DATABASE_URL` | Validated on startup but not yet used |
| `AUTH_CREDENTIALS` | Validated on startup but not yet used |
| `JWT_SECRET` | Validated on startup but not yet used |

## Acceptance Criteria

- [ ] `npm run dev` starts the server without errors
- [ ] `GET /health` returns `{ status: "ok" }` with HTTP 200
- [ ] Missing required env vars cause the process to exit with a clear error message before binding
- [ ] Sending a malformed JSON body to any future endpoint returns HTTP 400 via the error handler
- [ ] WebSocket connection to `ws://localhost:PORT` succeeds (no crash, no message exchange yet)
