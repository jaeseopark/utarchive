# PR 02 — Database Schema & Migrations

**Phase:** 1 — Foundation  
**Depends on:** PR 01

## Goal

Define the complete Postgres schema using Drizzle ORM and produce migration files that can be applied to any external Postgres instance.

## Scope

**In:**
- Drizzle ORM schema definitions for all 7 tables
- `drizzle-kit` migration generation
- GIN full-text search indexes
- Seed script for local development (optional, but recommended)

**Out:**
- No API routes
- No Drizzle query helpers used in routes (those come per-feature PR)
- No test database provisioning in CI (PR 05 / 20)

## Tables

| Table | Notes |
|---|---|
| `artists` | `id`, `name`, `aliases text[]`, `description`, `urls jsonb`, `created_at` |
| `songs` | `id`, `title`, `parent_id`, `master_id`, `platform_id`, `archived_at`, `released_at`, `play_count`, `url`, `file_path`, `duration real`, `file_extension varchar(16)`, `file_size_bytes bigint`, `cover_art_id uuid`, `description`, `preferred`, `trim_start real`, `trim_end real`, FTS `tsvector` column; check `trim_start < trim_end` |
| `song_artists` | `song_id`, `artist_id`, `display_order int` — PK `(song_id, artist_id)`; min one row per song |
| `albums` | `id`, `title`, `artist_id`, `year_released`, `cover_art_id uuid`, `track_list jsonb`, `urls jsonb`, `created_at` |
| `cover_art` | `id`, `file_path`, `width int`, `height int`, `file_extension varchar(16)`, `file_size_bytes bigint`, `created_at` |
| `album_songs` | `album_id`, `song_id`, `track_number` — PK `(album_id, song_id)` |
| `playlists` | `id`, `name`, `created_at` |
| `playlist_songs` | `playlist_id`, `song_id`, `position` — PK `(playlist_id, position)` |

## Indexes

- `songs`: GIN index on generated `tsvector` column over `(title || ' ' || coalesce(description,'') || ' ' || coalesce(url,''))`
- `artists`: GIN index on `to_tsvector('english', name)`
- `albums`: GIN index on `to_tsvector('english', title)`
- `songs.master_id`: B-tree index (tree queries)
- `song_artists.song_id`: B-tree index
- `song_artists.artist_id`: B-tree index (reverse lookup: all songs by artist)
- `album_songs.song_id`: B-tree index (reverse lookup)
- `playlist_songs.playlist_id, position`: covered by PK

## Acceptance Criteria

- [ ] `npx drizzle-kit generate` produces migration SQL without errors
- [ ] `npx drizzle-kit push` applies migrations to a live Postgres instance without errors
- [ ] All foreign key constraints are present and correct
- [ ] GIN indexes exist on the three FTS columns
- [ ] `drizzle-zod` can derive Zod insert/select schemas from all table definitions
