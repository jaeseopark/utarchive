# PR 12 — Auth UI

**Phase:** 3 — Frontend Shell  
**Depends on:** PR 04, PR 11

## Goal

Implement the login page, session context, and protected route wrapper so that unauthenticated users are redirected to `/login` and authenticated state is available throughout the app.

## Scope

**In:**
- Login page with id, password, and 6-digit TOTP code fields
- Session context (`useSession` hook) wrapping the app
- On mount, calls `GET /auth/me` to restore session after page reload
- Protected route component: wraps all routes except `/login`; redirects to `/login` if not authenticated
- Logout button in the nav sidebar

**Out:**
- No multi-user or registration flows

## Login Flow

1. User fills id + password + TOTP code and submits.
2. Frontend calls `POST /api/auth/login`.
3. On success: session context marks user as authenticated, navigate to `/artists`.
4. On 401: display an inline error message ("Invalid credentials or TOTP code").
5. `GET /auth/me` on app load: if 200 → restore session; if 401 → stay on `/login`.

## Session Context

```ts
interface SessionContext {
  user: { id: string } | null;
  isLoading: boolean;
  login(credentials: LoginPayload): Promise<void>;
  logout(): Promise<void>;
}
```

## Acceptance Criteria

- [ ] Visiting any protected route while unauthenticated redirects to `/login`
- [ ] Successful login redirects to `/artists`
- [ ] Wrong credentials show an error message without crashing
- [ ] Page refresh retains the session (cookie + `GET /auth/me` on mount)
- [ ] Logout clears the session and redirects to `/login`
- [ ] TOTP field accepts only 6 numeric digits
