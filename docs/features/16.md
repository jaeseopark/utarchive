# PR 16 — Album Pages

**Phase:** 4 — Song & Album Views  
**Depends on:** PR 08, PR 11, PR 12, PR 14

## Goal

Build the album list page and album detail page. The detail page merges the reference track list with registered songs and shows the full family tree for each registered track.

## Pages

### Album List — `/albums`

- Table of all albums: Title, Artist, Year Released
- Each row links to the album detail page
- Paginated (50 per page)

### Album Detail — `/albums/:id`

**Album header:**
- Title, Artist (linked), Year Released
- URL map as labeled links (same `UrlMap` component from PR 13)

**Track list table:**

Columns: **#**, **Title (reference)**, **Registered Song**, **Family Tree Toggle**

| # | Reference Title | Registered Song | |
|---|---|---|---|
| 1 | Song One | [Song One ↗](/songs/abc) | [▶ show tree] |
| 2 | Song Two | *(not registered)* | — |
| 3 | — | [Bonus Track ↗](/songs/xyz) | [▶ show tree] |

- Rows in `album_songs` but not in `track_list` are appended at the end with no reference title.
- Rows in `track_list` with no matching `album_songs` entry show "*(not registered)*" and no tree toggle.

**Inline family tree:**
- Clicking **[▶ show tree]** expands an inline `FamilyTree` component (from PR 14) below that row.
- Only one tree is expanded at a time (clicking another collapses the previous).
