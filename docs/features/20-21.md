## Docker Volume for Audio Files

The `file_path` column stores absolute paths inside the container. Recommended convention:

```bash
docker run \
  -v /host/music:/music \
  --env-file .env \
  utarchive
```

Store `file_path` values as `/music/artist/song.mp3` etc.

## Analytics Table

| Field | Type | Description |
|---|---|---|
| `id` | uuid PK | Event identifier |
| `song_id` | uuid FK songs(id) | Song listened to |
| `user_id` | text | Authenticated user id |
| `started_at` | timestamptz | When playback started |
| `duration_seconds` | real | Actual listened duration |
| `playback_percent` | real | Fraction of track listened (0-100) |
| `user_agent` | text nullable | Optional client user agent |

## Routes

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/api/analytics/listen` | ✅ | Log a listening session |
| `GET` | `/api/admin/analytics/stats` | ✅ | Return total analytics records |
| `POST` | `/api/admin/analytics/flush` | ✅ | Flush all analytics records |
| `POST` | `/api/admin/analytics/prune` | ✅ | Remove records older than the requested number of days |

## Payload

`POST /api/analytics/listen`

```ts
{
  songId: string,
  startedAt: string,        // ISO 8601
  durationSeconds: number,
  playbackPercent: number,  // 0-100
  userAgent?: string
}
```

> Note: `playbackPercent` should reflect the fraction of the playable segment consumed, not necessarily the full audio file duration.

## Actual Implementation Status

### Backend
- `backend/src/db/schema.ts` defines the `listening_analytics` table and related fields.
- `backend/src/db/queries/analytics.ts` includes insert, delete-all, prune, and count functions.
- `backend/src/routes/analytics.ts` exposes `POST /api/analytics/listen` with auth and zod validation.
- `backend/src/routes/admin/analytics.ts` exposes admin endpoints for stats, flush, and prune.
- `backend/src/routes/index.ts` registers analytics routes in the app router.
- `backend/src/middleware/requireAuth.ts` protects analytics routes, so unauthenticated access should be blocked.

### Validation
- `playbackPercent` is constrained to `0-100` in request validation.
- `durationSeconds` is non-negative.
- `songId` is validated as a UUID.
- `startedAt` accepts ISO 8601 datetimes with or without offset.

### Deployment
- `Dockerfile` is a multi-stage build that installs dependencies and builds the backend.
- `backend/src/app.ts` serves `frontend/dist` when present and falls back to `index.html` for SPA routing.
- `.nvmrc` is set to `24.17.0`, matching the Node base image in `Dockerfile`.
- `docker-compose-dev.yml` exists for local development, but there is no final `docker-compose.yml` in the repo.

### Documentation
- `README.md` already documents the core env vars, auth model, and Docker-oriented deployment strategy.
