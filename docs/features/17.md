# PR 17 — Streaming & Audio Player

**Phase:** 5 — Streaming & Playlists  
**Depends on:** PR 07, PR 10, PR 11, PR 12

## Goal

Implement the backend streaming endpoint and the frontend audio player component with preferred-song skip logic and a circuit breaker.

## Backend — Streaming Endpoint

`GET /api/stream/:songId`

- Requires valid session cookie (auth middleware).
- Looks up `file_path` from the `songs` table. Returns 404 if song not found or `file_path` is null.
- Validates that the file exists on disk. Returns 404 if missing.
- Reads the `Range` request header and responds with:
  - **206 Partial Content** + `Content-Range` header if a range is requested.
  - **200 OK** + full content if no range is specified.
- Sets `Content-Type` based on file extension: `audio/mp4` (mp4/m4a), `audio/mpeg` (mp3).
- Includes `X-Trim-Start`, `X-Trim-End`, `X-Duration`, and `X-File-Extension` response headers when applicable.
- When `Range: bytes=0-*` is detected (stream start), calls the play-count increment query (same as `POST /api/songs/:id/play`).

**Security:** path traversal prevention — `file_path` from the database is used as-is (the backend controls what paths are stored; no user-controlled path components are accepted in the streaming request).

## Trim & Resave

`POST /api/songs/:id/trim`

- Requires valid session cookie.
- Accepts `{ trimStart?: number, trimEnd?: number }` in the request body.
- Validates `0 <= trimStart < trimEnd <= duration`.
- Uses a server-side audio processing library to create a new trimmed file copy from the original `file_path`.
- Updates the song row with the new `file_path`, `duration`, `file_extension`, and `file_size_bytes` derived from the trimmed output.
- Resets `trim_start` and `trim_end` to null after a successful save.
- Returns the updated song metadata and trimmed file info.

If the song already has stored file metadata, they are replaced with values from the trimmed copy.

## Frontend — Audio Player Component

A persistent player bar fixed at the bottom of the layout. Visible only when a song is loaded.

### Controls
- Play / Pause
- Previous / Next (when in playlist context)
- Seek bar (uses byte-range seeking)
- Current time / Duration
- Volume slider

### Standalone Play (single song)

Triggered by the **"Play"** button on `/songs/:id`. No skip logic — plays the single song.

**Trim behaviour:** if the song payload includes `trimStart`, the audio element seeks to that position immediately after the `loadedmetadata` event. If `trimEnd` is set, the player monitors `timeupdate` and pauses/advances when `currentTime >= trimEnd`.

### Playlist Play Mode

Triggered from PR 18's playlist page. Player receives an ordered array of songs and a starting index.

**Preferred-skip logic:**
1. When a song ends (or immediately on load if `preferred = false`), check the next song.
2. If the next song is `preferred: false`, skip it and increment a `skipCount` counter.
3. If `skipCount >= playlist.songs.length`, **circuit breaker fires**: reset `skipCount` to 0 and play the current song regardless of `preferred` flag.
4. Reset `skipCount` to 0 whenever a `preferred: true` song is reached.

## Acceptance Criteria

### Backend
- [ ] `GET /api/stream/:songId` returns 206 with correct `Content-Range` for a range request
- [ ] `GET /api/stream/:songId` returns 404 if `file_path` is null
- [ ] `GET /api/stream/:songId` returns 401 without a session cookie
- [ ] `X-Trim-Start` and `X-Trim-End` headers are present when trim values are set; absent otherwise
- [ ] Play count increments exactly once per full stream start (range start = 0)
- [ ] Requesting a byte range in the middle does **not** increment play count
- [ ] `POST /api/songs/:id/trim` creates a trimmed copy and updates `file_path`, `duration`, `file_extension`, and `file_size_bytes`
- [ ] `trim_start` and `trim_end` are cleared after a successful trim save

### Frontend
- [ ] Audio player renders at the bottom of the layout when a song is loaded
- [ ] Seek bar moves to the clicked position and streams from that byte offset
- [ ] Audio player seeks to `trimStart` on load when the value is present
- [ ] Playback stops (and advances to next song in playlist mode) when `currentTime` reaches `trimEnd`
- [ ] A song with `preferred: false` is skipped immediately in playlist mode
- [ ] Circuit breaker plays the non-preferred song after cycling through all songs without finding a preferred one
- [ ] Player is hidden when no song is loaded
