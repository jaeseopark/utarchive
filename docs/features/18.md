# PR 18 — Playlist UI

**Phase:** 5 — Streaming & Playlists  
**Depends on:** PR 09, PR 11, PR 12, PR 17

## Goal

Build the playlist list and detail pages, including the ability to create/edit/delete playlists, manage songs within them, and launch playlist playback.

## Pages

### Playlist List — `/playlists`

- List of all playlists: Name, Song Count
- **"New Playlist"** button → opens a modal to enter a name and create
- Each row links to the playlist detail page

### Playlist Detail — `/playlists/:id`

**Header:**
- Playlist name (inline editable — click to edit, Enter to save via `PATCH /api/playlists/:id`)
- **"Play All"** button → passes the full ordered song list to `PlayerContext`
- **"Delete Playlist"** button (with confirmation dialog)

**Song List:**
- Ordered table: Position, Title, Artist, Preferred badge, Duration (if available), Remove button
- **Drag-to-reorder** rows (or up/down arrow buttons as a fallback) → triggers `PUT /api/playlists/:id/songs`
- **"Add Songs"** button → opens a search/browse modal to find and append songs

### Add Songs Modal

- Search input (calls `GET /api/search?q=`) to find songs by title or artist
- Results list with an **"Add"** button per song
- Already-in-playlist songs are shown with a checkmark and disabled button
- Closes on Escape or backdrop click

## Acceptance Criteria

- [ ] Creating a playlist from the list page navigates to the new playlist detail page
- [ ] Renaming a playlist inline saves on Enter and cancels on Escape
- [ ] Removing a song from the playlist updates the list without a full page reload
- [ ] Reordering songs persists after page refresh
- [ ] **"Play All"** starts the audio player from position 0 with the full playlist
- [ ] "Play All" respects preferred-skip logic (implemented in PR 17's PlayerContext)
- [ ] Add Songs modal shows search results in real time as the user types
- [ ] Songs already in the playlist are disabled in the modal
- [ ] Deleting a playlist requires confirmation and redirects to `/playlists` after success
