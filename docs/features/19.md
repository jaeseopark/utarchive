# PR 19 — Search UI

**Phase:** 6 — Search & Polish  
**Depends on:** PR 10, PR 11, PR 12

## Goal

Add a global search bar to the navigation and a dedicated search results page that surfaces songs, artists, and albums in a unified view.

## Components

### Global Search Bar (Nav)

- Text input in the top navigation area, always visible when authenticated
- Pressing Enter (or clicking a search icon) navigates to `/search?q=<query>`
- Does **not** do live/inline search (to keep it simple)

### Search Results Page — `/search?q=`

Three sections rendered vertically: **Songs**, **Artists**, **Albums**. Each section:
- Shows up to 20 results (the API maximum per category)
- Section header shows the category name and result count
- Empty sections are hidden entirely (not shown as "No results in Artists")
- If all three sections are empty, show a single "No results for '…'" message

**Song result row:** Title (linked to `/songs/:id`), Artist name, Released date, Preferred badge  
**Artist result row:** Name (linked to `/artists/:id`), Aliases  
**Album result row:** Title (linked to `/albums/:id`), Artist name, Year  

### URL Synchronisation

The search query is read from and written to the URL (`?q=`). Navigating back/forward in the browser restores the previous query and results.

## Acceptance Criteria

- [ ] Typing a query and pressing Enter navigates to `/search?q=<query>`
- [ ] Search results page shows correct results per category
- [ ] Empty categories are not rendered
- [ ] All result items link to the correct detail pages
- [ ] Navigating back restores the previous search query and results
- [ ] Loading state is shown while the API call is in flight
- [ ] Query of only whitespace is trimmed and treated as empty (no API call; show prompt to enter a query)
