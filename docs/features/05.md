# PR 05 — Dockerfile

**Phase:** 1 — Foundation  
**Depends on:** PR 03, PR 04 (server must be runnable)

## Goal

Package the entire stack into a single production Docker image. The React frontend is compiled during the build and its static output is served by the Express backend.

## Scope

**In:**
- Multi-stage Dockerfile:
  1. **Stage 1 (deps):** Install all workspace deps
  2. **Stage 2 (frontend-build):** `npm run build` in `frontend/`
  3. **Stage 3 (backend-build):** `tsc` in `backend/`
  4. **Stage 4 (production):** Copy compiled backend + frontend dist; install only production deps; set `CMD`
- `express-static` (or equivalent) middleware in the backend to serve `frontend/dist`
- `docker-compose.yml` for local dev (app + postgres containers)
- `.dockerignore`

**Out:**
- No CI/CD pipeline (can be added to PR 20)
- No nginx — Express serves static files directly
- No volume mounts for audio files (file_path refers to a path inside the container; the operator is responsible for mounting a volume at the right path)

## Static File Serving

The Express server serves the compiled React app from `frontend/dist` at `/`. All `/api/*` routes are handled first; everything else falls through to `index.html` (client-side routing support).

```
GET /api/*       → Express routes
GET /*           → serve frontend/dist/index.html (SPA fallback)
```

## Acceptance Criteria

- [ ] `docker build -t utarchive .` completes without errors
- [ ] `docker run --env-file .env utarchive` starts the server
- [ ] `GET /health` returns 200 from inside the container
- [ ] React app is served at `GET /` (returns HTML)
- [ ] `docker-compose up` starts app + postgres, health check passes
- [ ] Image contains no dev dependencies (`node_modules` is production-only in final stage)
