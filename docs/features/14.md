# PR 14 — Song Detail Page

**Phase:** 4 — Song & Album Views  
**Depends on:** PR 07, PR 11, PR 12

## Goal

Build the song detail page, including full metadata display and the family tree view.

## Page — `/songs/:id`

### Metadata Section

| Field | Display |
|---|---|
| Title | H1 heading |
| Artists | Each artist linked to `/artists/:id`; comma-separated if multiple |
| Released | Formatted date |
| Archived | Formatted date |
| Platform ID | Plain text (if present) |
| External URL | Clickable link (if present) |
| Play Count | Integer |
| Preferred | Badge: "Preferred" (green) or "Skip" (grey) |
| Trim Range | "0:15 → 3:42" formatted as `mm:ss`; hidden if neither value is set |
| Description | Preformatted multiline text block |

### Family Tree Section

Displayed as an indented table with columns: **Depth**, **Title**, **Artists**, **Released**, **Preferred**.

Example layout:
```
┌─────────────────────────────────────────────────────────────┐
│ Depth │ Title                   │ Artist  │ Released │ Pref │
├─────────────────────────────────────────────────────────────┤
│  0    │ Original Song [current] │ ArtistA │ 2020-01  │ ✓   │
│  1    │   ├─ Cover A            │ ArtistB │ 2021-03  │ ✓   │
│  2    │   │   └─ Cover of Cover │ ArtistC │ 2022-05  │ –   │
│  1    │   └─ Remix              │ ArtistD │ 2021-07  │ ✓   │
└─────────────────────────────────────────────────────────────┘
```

- The **current song** row is highlighted.
- Title cells use left-padding proportional to depth to show hierarchy visually without a graph library.
- Each title links to the respective `/songs/:id` page.
- Data comes from `GET /api/songs/:id/tree`.

### Action Bar

- **"Add child"** button → navigates to `/songs/new?parentId=:id` (implemented in PR 15)
- **"Play"** button → (disabled until PR 17)

## Acceptance Criteria

- [ ] All metadata fields render correctly; optional fields are hidden when null/empty
- [ ] Family tree shows the full tree even when viewing a leaf node
- [ ] Current song row is visually distinct (bold or highlighted background)
- [ ] Title cell indentation reflects depth correctly
- [ ] Clicking any title in the tree navigates to that song's detail page
- [ ] Tree with a single node (no parent, no children) renders a single row
- [ ] Loading and error states are handled
