# PR 07 — Song API

**Phase:** 2 — Core REST API  
**Depends on:** PR 02, PR 03, PR 04

## Goal

Expose full CRUD for songs and a tree endpoint that returns the entire family tree for a given song (from its master root down through all descendants).

## Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/songs` | ✅ | List songs (paginated, filterable) |
| `POST` | `/api/songs` | ✅ | Create a song |
| `GET` | `/api/songs/:id` | ✅ | Get a single song |
| `PATCH` | `/api/songs/:id` | ✅ | Update a song (partial) |
| `GET` | `/api/songs/:id/tree` | ✅ | Full family tree from master root |

## Request / Response Schemas

**POST / PATCH body:**
```ts
{
  title: string (max 500),
  parentId?: string | null,
  platformId?: string | null,
  releasedAt?: string,        // ISO 8601
  artistIds: string[],        // min length 1
  url?: string,
  filePath?: string,
  coverArtId?: string | null,
  description?: string,
  preferred?: boolean,         // default true
  trimStart?: number | null,  // seconds; must be < trimEnd if both set
  trimEnd?: number | null     // seconds
}
```
`masterId` is derived automatically: same as `id` if no `parentId`, else copy from parent.

**GET /songs/:id/tree response:**
```ts
{
  masterId: string,
  nodes: Array<{
    id: string,
    title: string,
    parentId: string | null,
    depth: number,             // 0 = master
    artistIds: string[],       // ordered by display_order
    coverArtId: string | null,
    preferred: boolean,
    playCount: number,
    releasedAt: string | null,
    trimStart: number | null,
    trimEnd: number | null
  }>
}
```
Nodes are ordered depth-first (master first).

## Tree Query Implementation

Recursive CTE anchored at `master_id`:
```sql
WITH RECURSIVE tree AS (
  SELECT *, 0 AS depth FROM songs WHERE id = $masterId
  UNION ALL
  SELECT s.*, t.depth + 1 FROM songs s
  JOIN tree t ON s.parent_id = t.id
)
SELECT * FROM tree ORDER BY depth, released_at;
```

## Query Filters (`GET /api/songs`)

- `?artistId=` — filter by artist
- `?masterId=` — filter to members of one family tree
- `?preferred=true|false`
- `?limit=` / `?offset=`

## Acceptance Criteria

- [ ] `POST /api/songs` with no `parentId` sets `masterId = id`
- [ ] `POST /api/songs` with a `parentId` sets `masterId` from the parent row
- [ ] `GET /api/songs/:id/tree` returns all descendants and ancestors in depth-first order
- [ ] `GET /api/songs/:id/tree` for a leaf node still returns the full tree (walks up to master first)
- [ ] `PATCH /api/songs/:id` cannot change `masterId` or `parentId` directly (those are structural; tree rewiring is out of scope)
- [ ] All endpoints return 401 without a valid session cookie
