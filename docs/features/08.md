# PR 08 — Album API

**Phase:** 2 — Core REST API  
**Depends on:** PR 02, PR 03, PR 04

## Goal

Expose CRUD for albums and endpoints to manage the album-song association table (add/remove songs, set track numbers).

## Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/albums` | ✅ | List albums (paginated) |
| `POST` | `/api/albums` | ✅ | Create an album |
| `GET` | `/api/albums/:id` | ✅ | Get album with merged track list |
| `PATCH` | `/api/albums/:id` | ✅ | Update album metadata |
| `PUT` | `/api/albums/:id/songs/:songId` | ✅ | Add or update a song on the album |
| `DELETE` | `/api/albums/:id/songs/:songId` | ✅ | Remove a song from the album |

## Request / Response Schemas

**POST / PATCH album body:**
```ts
{
  title: string (max 500),
  artistId: string,
  yearReleased?: number,
  coverArtId?: string | null,
  trackList?: Array<{ number: number, title: string }>,
  urls?: Record<string, string>
}
```

**PUT /albums/:id/songs/:songId body:**
```ts
{ trackNumber: number }
```

**GET /albums/:id response:**
```ts
{
  id: string,
  title: string,
  artistId: string,
  yearReleased: number | null,
  coverArtId: string | null,
  trackList: Array<{ number: number, title: string }>,    // from JSONB column
  urls: Record<string, string>,
  tracks: Array<{                                          // merged view
    trackNumber: number,
    song: { id: string, title: string } | null,          // null = not yet registered
    isRegistered: boolean
  }>
}
```

The `tracks` array is built by merging `track_list` (JSONB reference) with rows from `album_songs`. Track numbers present in `album_songs` but absent from `track_list` are appended. Unregistered reference tracks have `song: null`.

## Acceptance Criteria

- [ ] `POST /api/albums` creates an album and returns it with 201
- [ ] `GET /api/albums/:id` merges `track_list` and `album_songs` correctly
- [ ] A track in `album_songs` but not in `track_list` still appears in `tracks`
- [ ] `DELETE /api/albums/:id/songs/:songId` removes the association without deleting the song
- [ ] `PATCH /api/albums/:id` with `urls` replaces the entire JSONB map
- [ ] All endpoints return 401 without a valid session cookie
