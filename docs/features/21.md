# PR 21 — Listening Analytics

**Phase:** 6 — Search & Polish  
**Depends on:** PR 10, PR 17

## Goal

Record listening analytics for each playback session, including start time, duration listened, whether the user scrubbed, and the percent of the track consumed. Provide admin controls to flush or expire old analytics data.

## Scope

**In:**
- New analytics table for listening sessions
- Backend logging of play sessions whenever a user starts or stops playback
- Admin endpoints to flush analytics and prune records older than `ANALYTICS_RETENTION_DAYS`
- Optional retention setting via env var
- No frontend analytics dashboard yet; only admin API and data capture

**Out:**
- No live charting UI (can be added later)
- No user-facing analytics reporting
- No ML or advanced engagement scoring

## Analytics Table

| Field | Type | Description |
|---|---|---|
| `id` | uuid PK | Event identifier |
| `song_id` | uuid FK songs(id) | Song listened to |
| `user_id` | text | Authenticated user id |
| `started_at` | timestamptz | When playback started |
| `ended_at` | timestamptz | When playback ended |
| `duration_seconds` | real | Actual listened duration |
| `playback_percent` | real | Fraction of track listened (0-100) |
| `seeked` | boolean | Whether the user scrubbed during playback |
| `user_agent` | text nullable | Optional client user agent |
| `created_at` | timestamptz default now() | Event ingestion timestamp |

## Routes

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/api/analytics/listen` | ✅ | Log a listening session |
| `GET` | `/api/admin/analytics/stats` | ✅ | Return totals and retention stats |
| `POST` | `/api/admin/analytics/flush` | ✅ | Flush all analytics records |
| `POST` | `/api/admin/analytics/prune` | ✅ | Remove records older than `ANALYTICS_RETENTION_DAYS` |

## Payload

`POST /api/analytics/listen`

```ts
{
  songId: string,
  startedAt: string,        // ISO 8601
  endedAt: string,          // ISO 8601
  durationSeconds: number,
  playbackPercent: number,  // 0-100
  seeked: boolean,
  userAgent?: string
}
```

> Note: `playbackPercent` should reflect the fraction of the playable segment consumed, not necessarily the full audio file duration. For example, if a 60-second track only exposes 30 seconds of playable audio and the user listened to that entire segment, it should be reported as `100%` playback.

## Environment Variables

- `ANALYTICS_RETENTION_DAYS` — optional integer; when set, the prune endpoint removes analytics older than this many days.

## Acceptance Criteria

- [ ] `POST /api/analytics/listen` persists analytics records for each tracked playback session
- [ ] `GET /api/admin/analytics/stats` returns total records and current retention setting
- [ ] `POST /api/admin/analytics/flush` deletes all analytics records
- [ ] `POST /api/admin/analytics/prune` deletes records older than `ANALYTICS_RETENTION_DAYS`
- [ ] All analytics endpoints return 401 without a valid session cookie
- [ ] `playbackPercent` is constrained to 0-100 in request validation
