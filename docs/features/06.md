# PR 06 — Artist API

**Phase:** 2 — Core REST API  
**Depends on:** PR 02, PR 03, PR 04

## Goal

Expose full CRUD for artists and a convenience endpoint that lists all songs associated with an artist.

## Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/artists` | ✅ | List all artists (paginated) |
| `POST` | `/api/artists` | ✅ | Create an artist |
| `GET` | `/api/artists/:id` | ✅ | Get a single artist |
| `PATCH` | `/api/artists/:id` | ✅ | Update an artist (partial) |
| `GET` | `/api/artists/:id/songs` | ✅ | List all songs where `artist_id = :id` |

## Request / Response Schemas (Zod)

**POST / PATCH body:**
```ts
{
  name: string (max 255),
  aliases?: string[],
  description?: string,
  urls?: Record<string, string>
}
```

**GET /artists response item:**
```ts
{
  id: string,
  name: string,
  aliases: string[],
  description: string | null,
  urls: Record<string, string>,
  createdAt: string // ISO 8601
}
```

**GET /artists/:id/songs response:** array of song summary objects (id, title, releasedAt, preferred, playCount).

## Pagination

`GET /api/artists` accepts `?limit=50&offset=0`. Default limit: 50, max: 200.

## Acceptance Criteria

- [ ] `POST /api/artists` creates a record and returns it with 201
- [ ] `GET /api/artists/:id` returns 404 for unknown id
- [ ] `PATCH /api/artists/:id` updates only the provided fields
- [ ] `GET /api/artists/:id/songs` returns only songs belonging to that artist
- [ ] All endpoints return 401 without a valid session cookie
- [ ] Zod validation rejects a body with `name` exceeding 255 characters
