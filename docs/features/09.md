# PR 09 — Playlist API

**Phase:** 2 — Core REST API  
**Depends on:** PR 02, PR 03, PR 04

## Goal

Expose CRUD for playlists and endpoints to add, remove, and reorder songs within a playlist.

## Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/playlists` | ✅ | List all playlists |
| `POST` | `/api/playlists` | ✅ | Create a playlist |
| `GET` | `/api/playlists/:id` | ✅ | Get playlist with ordered song list |
| `PATCH` | `/api/playlists/:id` | ✅ | Rename a playlist |
| `DELETE` | `/api/playlists/:id` | ✅ | Delete a playlist (not the songs) |
| `POST` | `/api/playlists/:id/songs` | ✅ | Append a song to the playlist |
| `DELETE` | `/api/playlists/:id/songs/:songId` | ✅ | Remove a song from the playlist |
| `PUT` | `/api/playlists/:id/songs` | ✅ | Replace the full ordered song list |

## Request / Response Schemas

**POST /playlists body:**
```ts
{ name: string (max 255) }
```

**POST /playlists/:id/songs body:**
```ts
{ songId: string, position?: number }  // omit position to append at end
```

**PUT /playlists/:id/songs body (full reorder):**
```ts
{ songIds: string[] }  // new ordered list; must contain same set of songs currently in playlist
```

**GET /playlists/:id response:**
```ts
{
  id: string,
  name: string,
  createdAt: string,
  songs: Array<{
    position: number,
    song: { id: string, title: string, preferred: boolean, filePath: string | null }
  }>
}
```

## Ordering Implementation

`playlist_songs` uses a 0-based integer `position` column. Insertions at a specific position shift existing entries with `position >= target` up by 1. Deletion shifts down. Full reorder (`PUT`) replaces all rows in one transaction.

## Acceptance Criteria

- [ ] Songs in `GET /playlists/:id` are returned in ascending `position` order
- [ ] `POST /playlists/:id/songs` without `position` appends to end
- [ ] `POST /playlists/:id/songs` with `position: 0` inserts at beginning and shifts existing songs
- [ ] `PUT /playlists/:id/songs` with a different-length array returns 400
- [ ] `DELETE /playlists/:id` removes the playlist and all `playlist_songs` rows (cascade)
- [ ] A song can appear in multiple playlists simultaneously
- [ ] All endpoints return 401 without a valid session cookie
