## Goal

Implement the new song form and the "add child" flow. The add-child flow pre-populates most fields from the parent song, lets the user edit them, then saves on confirmation.

## Pages / Flows

### New Song â€” `/songs/new`

Standard form for creating a root (master) song with all fields editable from scratch.

### Add Child â€” `/songs/new?parentId=:id`

When `parentId` is in the query string:
1. Fetch the parent song via `GET /api/songs/:parentId`.
2. Pre-populate the form with values copied from the parent (see field copy rules below).
3. Show a banner: "Adding a child of: **[Parent Title]**".
4. User edits fields as needed and clicks **Save**.
5. On success: navigate to the new song's detail page.

### Edit Song â€” `/songs/:id/edit`

Same form as new song, pre-populated from the existing song. Uses `PATCH /api/songs/:id`.

## Field Copy Rules (Add Child)

| Field | Behaviour |
|---|---|
| `title` | Copied as-is (user should edit to differentiate) |
| `artistIds` | Copied (collaborators often same; user can change) |
| `releasedAt` | Cleared (child has its own release date) |
| `platformId` | Cleared |
| `url` | Cleared |
| `filePath` | Cleared |
| `trimStart` | Cleared |
| `trimEnd` | Cleared |
| `description` | Cleared |
| `preferred` | Copied (default true) |
| `parentId` | Set to parent's `id` |
| `masterId` | Derived by backend |

## Form Fields

All fields use shadcn/ui form primitives with React Hook Form + Zod resolver.

| Field | Input type | Validation |
|---|---|---|
| Title | Text | Required, max 500 |
| Artists | Multi-select combobox (search artists) | At least one required |
| Released at | Date picker | Optional |
| Platform ID | Text | Optional, max 255 |
| External URL | URL input | Optional |
| File path | Text | Optional |
| Trim Start | Number input (seconds) | Optional; must be < Trim End if both set |
| Trim End | Number input (seconds) | Optional; must be > Trim Start if both set |
| Description | Textarea | Optional |
| Preferred | Toggle / Checkbox | Default true |
