## Frontend Developement

### Testing

Include a storybook test for each component being added or modified.

The storybook test file should provide playbook coverage that verifies both appearance and behavior.

1. For the user to preview the component
2. Basic `play` test to see if the elements render as expected
3. Every interaction path covered by a `play` test
4. A clear story-level example for each component state or variant
5. Regression coverage for any user-facing behavior changes
